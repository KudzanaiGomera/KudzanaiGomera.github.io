Analyst Strategic Triage Focus

To triage smarter, an analyst must prioritize Intent and Origin over raw logs. When a detection rule fires, focus on these three pillars:

  • The "Why": Is this activity part of a known business process or a deviation? (e.g., Software Center vs. Browser download).
  • The "Path": Trace the execution chain. Malicious activity usually starts from an external vector (web/email) and moves toward persistence.
  • The "Precedent": Has this user or host performed similar actions? Cross-reference with HR/Departmental roles to identify "Shadow IT" vs. actual compromise.

Intel: Rule_Monitoring Suspicious Execution of Autorun Registry Keys

Executive Summary

This P3 incident highlights a case of "Shadow IT" where a user bypassed corporate security controls to resolve a local productivity issue. The event began when our monitoring systems flagged a suspicious modification to the Windows AutoRun registry hive on a Marketing analyst’s workstation. Investigation revealed that the user, facing technical difficulties with the officially sanctioned versions of Notepad++ available in the Software Center and Coupa, sought an alternative solution via a public search engine. This led to the download and installation of a non-standard version of the software from an unverified third-party domain.

While the analysis confirmed the activity was user-initiated and lacked malicious intent (non-targeted), the installer contained Potentially Unwanted Programs (PUP/Grayware) and established a persistence mechanism via the registry. This constitutes a direct violation of the Corporate Acceptable Use Policy. The risk was mitigated by initiating an automated software removal, and the user has been provided with educational resources regarding the risks of third-party repositories. The incident is classified as Resolved | Policy Non-Compliance (Bad BAU).

Alert Details

Microsoft Defender for Endpoint (MDE) identified an unauthorized modification to the AutoRun registry keys—a common technique used by persistent threats to ensure software execution upon system startup. In this instance, the alert was triggered on JD-MKT-001 (User: CORP\jdoe) during the installation of a non-standard software package.

Raw command line string observed:
Process execution originated via Chrome download; direct registry injection observed from setup binary.

Endpoint & Network Analysis (Splunk + MDE Insights)
  • Event Sequence: The narrative begins with chrome.exe accessing freefiles.download. A 2.4MB executable, notepadplus_setup.exe, was downloaded to the user's local directory.
  • Execution Flow: explorer.exe > notepadplus_setup.exe. Upon execution, the installer immediately attempted to secure persistence to ensure the application remained active across reboots.
  • Registry Modification:
    • Key Hive: HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run
    • Value Name: NotepadPlus
    • Set Data: C:\Users\jdoe\AppData\Roaming\NotepadPlus\notepad++.exe
  • Post-Installation: No lateral movement, credential dumping, or C2 (Command & Control) beacons were observed. The activity remained localized to the user's profile.
IOC & Threat Intelligence
  • VirusTotal: SHA256 flagged by 7 engines; categorized as PUP/Grayware (Adware/Bundler).
  • Reputation: The source domain freefiles.download is currently blacklisted for hosting low-reputation software installers.
  • Sandbox Behavior: Joe Sandbox analysis indicates the file behaves as a generic installer with embedded telemetry, but no high-severity malicious payloads were detected.
Documentation & Business Justification
  • Official Channel: Notepad++ is a "Standard-Build" application available for one-click install via the Software Center.
  • User Rationale: The user stated the approved version "would not load plugins" required for their specific workflow, leading them to search for a "full version" online.
  • Compliance Gap: No prior ticket was submitted to IT Support to address the plugin issue before the user bypassed security protocols.
Conclusion
  • The user was debriefed on the inherent risks of downloading executables from unverified sources.
  • A high-priority Work Order was executed for the forensic removal of the unauthorized software: WO0026738637388.
  • The investigation confirms no active compromise; this was a "Bad BAU" scenario driven by a productivity gap.
  • Closing Action: Resolved | Bad BAU
